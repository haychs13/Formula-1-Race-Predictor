# Formula-1-Race-Predictor-
Machine learning web app that predicts Formula 1 podium finishes using driver performance, qualifying data and circuit history. Built by Hayder Sayyid
Predicts the probability of an F1 driver finishing on the podium (top 3), trained on 2021–2025 race data.

Make sure you have **Python** and **pip** installed. You can download Python from https://python.org — pip is included automatically.

---

## Windows (No Git Required)

1. Click the green **Code** button on this page → **Download ZIP**
2. Extract the ZIP — you'll get a folder inside a folder, go into the inner one
3. Open PowerShell and run:

```
cd "C:\Users\YourName\Downloads\F1-Race-Predictor-main\F1-Race-Predictor-main"
pip install -r requirements.txt
streamlit run app.py
```

4. Open **http://localhost:8501** in your browser

> Replace `YourName` with your actual Windows username.

---

## Clone (Git)

```
git clone https://github.com/haychs13/F1-Race-Predictor.git
cd F1-Race-Predictor
pip install -r requirements.txt
streamlit run app.py
```

Open **http://localhost:8501** in your browser.

## Tabs

- **Predict a Race** — pick a driver and circuit to get a podium probability, confidence score, and plain-English explanation of the prediction
- **Model Accuracy** — see how the model performed during training, including F1 score, precision, recall, model comparison chart, and confusion matrix
- **How It Works** — explanation of the confidence score, prediction process, data sources, all 13 features, and feature importance chart

## Disclaimer

This app uses F1 race data from the 2021–2025 seasons sourced from the Ergast F1 dataset. Data has not been updated beyond this period and driver line-ups, team names, and circuits may have changed. Predictions are generated by a machine learning model trained on historical patterns and are not guaranteed to reflect current or future performance. This project is intended solely as a machine learning demonstration and must not be used for gambling, betting, or any commercial purposes. For production use, this model could be extended with live data via the FastF1 or OpenF1 APIs and automated retraining after each race weekend.
